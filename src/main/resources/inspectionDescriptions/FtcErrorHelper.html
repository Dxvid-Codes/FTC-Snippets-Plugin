FTC Error Helper Inspection

Detects common FTC SDK programming errors and provides quick fixes to resolve them. This inspection helps FIRST Tech Challenge teams avoid common pitfalls in their robot code.

Detected Issues

Missing waitForStart()

Problem: The runOpMode() method doesn't call waitForStart()
Impact: Code may execute before the match starts, causing unpredictable behavior
Quick Fix: Automatically inserts waitForStart(); at the beginning of runOpMode()

Missing telemetry.update()

Problem: telemetry.addData() is used without calling telemetry.update()
Impact: Telemetry data won't appear on the Driver Station
Quick Fix: Automatically inserts telemetry.update(); after telemetry statements

Uninitialized Hardware Devices

Problem: Hardware devices (motors, servos, sensors) are declared but not initialized with hardwareMap.get()
Impact: NullPointerException when trying to use the hardware device
Quick Fix: Automatically generates initialization code in runOpMode() or init()

Gamepad Usage Issues

Problem: Gamepad button checks without proper conditional statements
Impact: Unintended continuous activation or missed button presses
Quick Fix: Suggests wrapping in if statements (manual fix required)

Controller Direction Issues

Problem: Potential inverted stick axes that may cause unintuitive controls
Impact: Confusing driver controls
Quick Fix: Information warning about common axis inversion patterns

Missing Annotations

Problem: TeleOp/OpMode classes missing required @TeleOp or @Autonomous annotations
Impact: OpMode won't appear in Driver Station selection list
Quick Fix: Warning to add appropriate annotation

Supported Hardware Types

  • DcMotor / DcMotorEx
  • Servo / CRServo
  • BNO055IMU (Rev Hub IMU)
  • DistanceSensor
  • ColorSensor
  • Other FTC SDK hardware devices

How to Use

The inspection runs automatically on Java files in your FTC project. When issues are detected:

  1. Look for highlighted code with warnings/errors
  2. Press Alt+Enter (or click the lightbulb) on the highlighted code
  3. Select the quick fix to automatically resolve the issue

Requirements

This inspection only activates for classes that extend FTC OpMode classes:

  • LinearOpMode
  • OpMode
  • Classes with "OpMode" in their name

Example Quick Fixes

Before:


public void runOpMode() {
    // Missing waitForStart()
    motor.setPower(1.0);
}

After Quick Fix:


public void runOpMode() {
    waitForStart();  // Automatically inserted
    motor.setPower(1.0);
}

Before:


DcMotor leftMotor;
// Not initialized with hardwareMap

public void runOpMode() {
    leftMotor.setPower(0.5);  // NullPointerException!
}

After Quick Fix:


DcMotor leftMotor;

public void runOpMode() {
    leftMotor = hardwareMap.get(DcMotor.class, "leftMotor");  // Automatically inserted
    leftMotor.setPower(0.5);
}

Troubleshooting

If the inspection isn't working:

  • Ensure your class extends an FTC OpMode class
  • Check that the FTC SDK is properly imported in your project
  • Verify the plugin is enabled in Settings/Preferences > Plugins
  • Make sure Java inspections are enabled
